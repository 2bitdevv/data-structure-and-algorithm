เนื้อหา: ขั้นตอนวิธีการเรียงลำดับ (Sorting Algorithms)
บทนำ: การเรียงลำดับ (Sorting) เป็นกระบวนการจัดเรียงข้อมูลให้อยู่ในลำดับตามที่ต้องการ เช่น เรียงจากน้อยไปมาก (Ascending) หรือจากมากไปน้อย (Descending) เป็นพื้นฐานสำคัญในการประมวลผลข้อมูล เพื่อให้การค้นหาและการวิเคราะห์ข้อมูลทำได้อย่างมีประสิทธิภาพ ในรายงานนี้นำเสนอขั้นตอนวิธีการเรียงลำดับ 6 แบบ พร้อมตัวอย่างและภาพประกอบการทำงาน

1. Selection Sort
แนวคิด: เลือกหาค่าที่น้อยที่สุดในส่วนที่ยังไม่ได้เรียงลำดับ แล้วนำมาไว้ตำแหน่งแรกสุดของส่วนนั้น ทำซ้ำจนกว่าข้อมูลทั้งหมดจะถูกเรียงลำดับ
ขั้นตอนการทำงาน
    1. แบ่งข้อมูลออกเป็น 2 ส่วน: ส่วนที่เรียงแล้ว (ด้านซ้าย) และส่วนที่ยังไม่เรียง (ด้านขวา)
    2. หาค่าที่น้อยที่สุดในส่วนที่ยังไม่เรียง
    3. สลับค่าที่น้อยที่สุดกับตัวแรกของส่วนที่ยังไม่เรียง
    4. ขยายส่วนที่เรียงแล้วไปทางขวา 1 ตำแหน่ง
    5. ทำซ้ำจนกว่าข้อมูลทั้งหมดจะถูกเรียง

Example: การทำงานข้อมูลเริ่มต้น: [64, 25, 12, 22, 11]
    รอบที่ 1: หาค่าน้อยสุดใน [64, 25, 12, 22, 11] → 11
            สลับ 64 กับ 11 → [11, 25, 12, 22, 64]
    รอบที่ 2: หาค่าน้อยสุดใน [25, 12, 22, 64] → 12
            สลับ 25 กับ 12 → [11, 12, 25, 22, 64]
    รอบที่ 3: หาค่าน้อยสุดใน [25, 22, 64] → 22
            สลับ 25 กับ 22 → [11, 12, 22, 25, 64]
    รอบที่ 4: หาค่าน้อยสุดใน [25, 64] → 25
            ไม่ต้องสลับ → [11, 12, 22, 25, 64]
    ผลลัพธ์: [11, 12, 22, 25, 64]

ข้อดี: ง่ายต่อการ implement, ไม่ใช้หน่วยความจำเพิ่ม
ข้อเสีย: ความเร็ว O(n²) เสมอ ไม่เหมาะกับข้อมูลขนาดใหญ่

ความซับซ้อน (Complexity)
    Best Case   |  O(n²)
    Average Case|  O(n²)
    Worst Case  |  O(n²)
    Space       |  O(1) 

*** หมายเหตุ Big O Notation ใช้บอกประสิทธิภาพของอัลกอริทึม โดยเฉพาะเวลา (Time Complexity) และหน่วยความจำ (Space Complexity) ของ Sort
    BEST	    กรณีดีที่สุด    ข้อมูลเรียงลำดับแล้ว (สำหรับ Bubble Sort ที่ optimize แล้ว)
    AVERAGE	    กรณีทั่วไป	   ข้อมูลสุ่ม
    WORST	    กรณีแย่ที่สุด   ข้อมูลเรียงกลับด้าน
    Example: O(n²) - Time Complexity
                n = จำนวนข้อมูลที่ประมวลผล
                n² = เวลาเพิ่มขึ้นแบบ ยกกำลังสอง
            ** ยิ่งข้อมูลมาก เวลาจะเพิ่มแบบ ทวีคูณ **
            SPACE: O(1) = ใช้หน่วยความจำคงที่ ไม่เพิ่มตามจำนวนข้อมูล เรียกว่า In-place algorithm
    อย่างเช่น ข้อมูล 10 รายการ → 100 รอบการทำงาน
            ข้อมูล 100 รายการ → 10,000 รอบการทำงาน
            ข้อมูล 1,000 รายการ → 1,000,000 รอบการทำงาน

2. Insertion Sort
แนวคิด: เปรียบเสมือนการเรียงไพ่ในมือ โดยหยิบการ์ดใบใหม่มาแทรกในตำแหน่งที่เหมาะสมในส่วนที่เรียงไว้แล้ว
ขั้นตอนการทำงาน
    1. เริ่มจากตัวที่ 2 (index 1) ถือเป็นตัวที่ต้องการแทรก (key)
    2. เปรียบเทียบ key กับตัวทางซ้ายมือ
    3. ถ้าตัวทางซ้ายมีค่ามากกว่า ให้เลื่อนตัวนั้นไปทางขวา
    4. ทำซ้ำจนกว่าจะเจอตำแหน่งที่เหมาะสม แล้วแทรก key
    5. ทำกับตัวถัดไปจนครบ

Example: การทำงานข้อมูลเริ่มต้น: [5, 2, 4, 6, 1, 3]
    รอบที่ 1: key = 2 เปรียบเทียบกับ [5]
             [5] > 2 → เลื่อน 5 ไปขวา → [5, 5, 4, 6, 1, 3]
            แทรก 2 → [2, 5, 4, 6, 1, 3]
    รอบที่ 2: key = 4 เปรียบเทียบกับ [2, 5]
             [5] > 4 → เลื่อน 5 ไปขวา → [2, 5, 5, 6, 1, 3]
             [2] < 4 → แทรก 4 → [2, 4, 5, 6, 1, 3]
    รอบที่ 3: key = 6 เปรียบเทียบกับ [2, 4, 5]
             [5] < 6 → ไม่ต้องเลื่อน → [2, 4, 5, 6, 1, 3]
    รอบที่ 4: key = 1 เปรียบเทียบกับ [2, 4, 5, 6]
            เลื่อนทั้งหมด  → [2, 4, 5, 6, 6]
                        → [2, 4, 5, 5, 6]
                        → [2, 4, 4, 5, 6]
                        → [2, 2, 4, 5, 6]
            แทรก 1 → [1, 2, 4, 5, 6, 3]
    รอบที่ 5: key = 3 เปรียบเทียบกับ [1, 2, 4, 5, 6]
            เลื่อน 4,5,6 → แทรก 3 → [1, 2, 3, 4, 5, 6]
    ผลลัพธ์: [1, 2, 3, 4, 5, 6]

ข้อดี: ทำงานเร็วเมื่อข้อมูลใกล้เคียงลำดับที่ต้องการ, เป็น Stable Sort
ข้อเสีย: ช้าเมื่อข้อมูลมีขนาดใหญ่

ความซับซ้อน (Complexity)
    Best Case   |  O(n) (เมื่อข้อมูลเรียงแล้ว)
    Average Case|  O(n²)
    Worst Case  |  O(n²)
    Space       |  O(1)

3. Bubble Sort
แนวคิด: เปรียบเทียบข้อมูลที่อยู่ติดกัน ถ้าตัวหน้ามากกว่าตัวหลัง ให้สลับที่กัน ทำให้ค่ามากๆ จะค่อยๆ "ลอย" ไปด้านขวาสุดเหมือนฟองอากาศ
ขั้นตอนการทำงาน
    1. เปรียบเทียบข้อมูลคู่ที่ติดกันตั้งแต่ซ้ายไปขวา
    2. ถ้าตัวซ้าย > ตัวขวา ให้สลับที่
    3. เมื่อจบรอบ ค่าที่มากที่สุดจะไปอยู่ขวาสุด
    4. ทำซ้ำโดยไม่ต้องสนใจค่าที่เรียงแล้วทางขวา
    5. ถ้าในรอบใดไม่มีการสลับ แสดงว่าข้อมูลเรียงแล้ว หยุดการทำงาน

Example: การทำงานข้อมูลเริ่มต้น: [5, 1, 4, 2, 8]
    รอบที่ 1:
        [5,1,4,2,8] → 5>1 สลับ → [1,5,4,2,8]
        [1,5,4,2,8] → 5>4 สลับ → [1,4,5,2,8]
        [1,4,5,2,8] → 5>2 สลับ → [1,4,2,5,8]
        [1,4,2,5,8] → 5<8 ไม่สลับ
        ได้ค่า max 8 ที่ตำแหน่งสุดท้าย
    รอบที่ 2:
        [1,4,2,5,8] → 1<4 ไม่สลับ
        [1,4,2,5,8] → 4>2 สลับ → [1,2,4,5,8]
        [1,2,4,5,8] → 4<5 ไม่สลับ
        ได้ค่า 5 ที่ตำแหน่งรองสุดท้าย
    รอบที่ 3:
        [1,2,4,5,8] → 1<2 ไม่สลับ
        [1,2,4,5,8] → 2<4 ไม่สลับ
        ไม่มีการสลับ → หยุดทำงาน
    ผลลัพธ์: [1, 2, 4, 5, 8]

ข้อดี: ง่ายที่สุด, เข้าใจง่าย, ตรวจสอบได้ว่าข้อมูลเรียงแล้วหรือยัง
ข้อเสีย: ช้ามาก, มีการสลับหลายครั้ง

ความซับซ้อน (Complexity)
    Best Case   |  O(n) (เมื่อข้อมูลเรียงแล้ว)
    Average Case|  O(n²)
    Worst Case  |  O(n²)
    Space       |  O(1)

4. Shell Sort
แนวคิด: เป็นการปรับปรุงจาก Insertion Sort โดยเปรียบเทียบและสลับข้อมูลที่อยู่ห่างกัน (gap) เพื่อให้ข้อมูลเคลื่อนที่ไปยังตำแหน่งที่ถูกต้องได้เร็วขึ้น แล้วค่อยๆ ลด gap ลงจนเป็น 1
ขั้นตอนการทำงาน
    1. กำหนด gap เริ่มต้น (มักใช้ n/2)
    2. เปรียบเทียบและสลับข้อมูลที่ห่างกันเท่ากับ gap
    3. เมื่อทำครบทุกชุดแล้ว ลด gap ลงครึ่งหนึ่ง
    4. ทำซ้ำจนกว่า gap = 1 (เป็นการทำ Insertion Sort ปกติ)

Example: การทำงานข้อมูลเริ่มต้น: [9, 8, 3, 7, 5, 6, 4, 1]
    gap = 4 (n/2):
    เปรียบเทียบตำแหน่งห่างกัน 4:
        [9,5] → 9>5 สลับ → [5,8,3,7,9,6,4,1]
        [8,6] → 8>6 สลับ → [5,6,3,7,9,8,4,1]
        [3,4] → 3<4 ไม่สลับ
        [7,1] → 7>1 สลับ → [5,6,3,1,9,8,4,7]
        ได้: [5,6,3,1,9,8,4,7]
    gap = 2:
    เปรียบเทียบห่างกัน 2:
        [5,3,9,4] → 5>3 สลับ → [3,6,5,1,9,8,4,7]
                    → 5<9 ไม่สลับ
                    → 9>4 สลับ → [3,6,4,1,5,8,9,7]
        [6,1,8,7] → 6>1 สลับ → [3,1,4,6,5,8,9,7]
                    → 4<8 ไม่สลับ
                    → 8>7 สลับ → [3,1,4,6,5,7,9,8]
        ได้: [3,1,4,6,5,7,9,8]
    gap = 1 (Insertion Sort):
    ทำการเรียงแบบ Insertion Sort:
    [3,1,4,6,5,7,9,8]   → [1,3,4,6,5,7,9,8]
                        → [1,3,4,5,6,7,9,8]
                        → [1,3,4,5,6,7,8,9]
    ผลลัพธ์: [1, 3, 4, 5, 6, 7, 8, 9]

ข้อดี: เร็วกว่า Insertion Sort สำหรับข้อมูลขนาดกลาง, ไม่ใช้หน่วยความจำเพิ่ม
ข้อเสีย: เลือก gap ที่เหมาะสมยาก, ไม่ค่อยเสถียร (Unstable)

ความซับซ้อน
    Best Case   | O(n log n)
    Average Case| O(n log² n) ถึง O(n^(3/2))
    Worst Case  | O(n²) (ขึ้นอยู่กับชุด gap)
    Space       | O(1)

5. Merge Sort
แนวคิด: ใช้หลักการ Divide and Conquer (แบ่งแยกแล้วเอาชนะ) โดยแบ่งข้อมูลครึ่งหนึ่งไปเรื่อยๆ จนเหลือตัวเดียว แล้วค่อยรวมกลับเข้าด้วยกันอย่างเป็นระเบียบ
ขั้นตอนการทำงาน
    1. แบ่งข้อมูลออกเป็น 2 ส่วนเท่าๆ กัน
    2. เรียกใช้ Merge Sort กับส่วนซ้ายและส่วนขวา (ทำซ้ำจนเหลือตัวเดียว)
    3. รวม (Merge) 2 ส่วนที่เรียงแล้วเข้าด้วยกัน:
        - เปรียบเทียบตัวหน้าของแต่ละส่วน
        - นำตัวที่น้อยกว่าใส่ในอาร์เรย์ผลลัพธ์
        - ทำซ้ำจนครบ

    Example: การทำงานข้อมูลเริ่มต้น: [38, 27, 43, 3, 9, 82, 10]
    ขั้นตอนที่ 1: แบ่งครึ่ง
        [38, 27, 43, 3] กับ [9, 82, 10]
    ขั้นตอนที่ 2: แบ่งต่อไปเรื่อยๆ
        [38, 27] [43, 3] [9, 82] [10]
        [38] [27] [43] [3] [9] [82] [10]
    ขั้นตอนที่ 3: รวมกลับ
        รวม [38] กับ [27] → [27, 38]
        รวม [43] กับ [3] → [3, 43]
        รวม [27, 38] กับ [3, 43] → [3, 27, 38, 43]

        รวม [9] กับ [82] → [9, 82]
        รวม [9, 82] กับ [10] → [9, 10, 82]
    ขั้นตอนที่ 4: รวมสองส่วนสุดท้าย
    [3, 27, 38, 43] กับ [9, 10, 82]
        เปรียบเทียบ 3 กับ 9 → [3]
        เปรียบเทียบ 27 กับ 9 → [3, 9]
        เปรียบเทียบ 27 กับ 10 → [3, 9, 10]
        เปรียบเทียบ 27 กับ 82 → [3, 9, 10, 27]
        เปรียบเทียบ 38 กับ 82 → [3, 9, 10, 27, 38]
        เปรียบเทียบ 43 กับ 82 → [3, 9, 10, 27, 38, 43]
        เหลือ 82 → [3, 9, 10, 27, 38, 43, 82]
    ผลลัพธ์: [3, 9, 10, 27, 38, 43, 82]

ข้อดี: ความเร็วคงที่ O(n log n) เสมอ, เหมาะกับข้อมูลขนาดใหญ่, เป็น Stable Sort
ข้อเสีย: ใช้หน่วยความจำเพิ่ม O(n) สำหรับอาร์เรย์ชั่วคราว

ความซับซ้อน
    Best Case   | O(n log n)
    Average Case| O(n log n)
    Worst Case  | O(n log n)
    Space       | O(n)

6. Quick Sort
แนวคิด: ใช้หลักการ Divide and Conquer เช่นกัน โดยเลือกค่าหนึ่งเป็น pivot แล้วจัดเรียงให้ค่าน้อยกว่า pivot ไปไว้ซ้าย ค่ามากกว่าไปไว้ขวา จากนั้นทำซ้ำกับส่วนซ้ายและขวา
ขั้นตอนการทำงาน
    1. เลือก pivot (นิยมเลือกตัวกลาง, ตัวแรก, ตัวสุดท้าย, หรือสุ่ม)
    2. Partition: จัดเรียงข้อมูลใหม่ให้:
        - ตัวที่น้อยกว่า pivot ไปอยู่ซ้าย
        - ตัวที่มากกว่า pivot ไปอยู่ขวา
        - pivot อยู่ตรงกลาง
    3. เรียกใช้ Quick Sort กับส่วนซ้าย (น้อยกว่า pivot)
    4. เรียกใช้ Quick Sort กับส่วนขวา (มากกว่า pivot)

Example: การทำงานข้อมูลเริ่มต้น: [3, 7, 8, 5, 2, 1, 9, 5, 4]
    เลือก pivot = 5 (ตำแหน่งกลาง)

    รอบที่ 1: Partition ด้วย pivot = 5
                ซ้าย (<5): [3, 2, 1, 4]
                กลาง (=5): [5, 5]
                ขวา (>5): [7, 8, 9]
            ได้: [3, 2, 1, 4, 5, 5, 7, 8, 9]

    รอบที่ 2: Quick Sort ส่วนซ้าย [3, 2, 1, 4]
            เลือก pivot = 2
            Partition:
                ซ้าย (<2): [1]
                กลาง (=2): [2]
                ขวา (>2): [3, 4]
            ได้: [1, 2, 3, 4]

    รอบที่ 3: Quick Sort ส่วนขวา [7, 8, 9]
            เลือก pivot = 8
            Partition:
                ซ้าย (<8): [7]
                กลาง (=8): [8]
                ขวา (>8): [9]
            ได้: [7, 8, 9]

    ผลลัพธ์: [1, 2, 3, 4, 5, 5, 7, 8, 9]

ข้อดี: เร็วมากในทางปฏิบัติ, ทำงานในที่เดิม (In-place) ไม่ต้องใช้หน่วยความจำมาก
ข้อเสีย: Worst Case O(n²) เมื่อเลือก pivot ไม่ดี, ไม่ Stable

ความซับซ้อน
    Best Case   | O(n log n)
    Average Case| O(n log n)
    Worst Case  | O(n²)
    Space       | O(log n) (สำหรับ recursive call stack)

สรุปเปรียบเทียบ
| อัลกอริทึม       | Best Case   | Average Case | Worst Case | Space    | Stability |
|--------------- |------------|---------------|------------|----------|----------|
| Selection Sort | O(n²)      | O(n²)         | O(n²)      | O(1)     | No       |
| Insertion Sort | O(n)       | O(n²)         | O(n²)      | O(1)     | Yes      |
| Bubble Sort    | O(n)       | O(n²)         | O(n²)      | O(1)     | Yes      |
| Shell Sort     | O(n log n) | O(n log² n)   | O(n²)      | O(1)     | No       |
| Merge Sort     | O(n log n) | O(n log n)    | O(n log n) | O(n)     | Yes      |
| Quick Sort     | O(n log n) | O(n log n)    | O(n²)      | O(log n) | No       |

การเลือกใช้อัลกอริทึม
    1. ข้อมูลขนาดเล็ก (n < 50): Insertion Sort ทำงานได้ดี
    2. ข้อมูลใกล้เคียงลำดับที่ต้องการ: Insertion Sort หรือ Bubble Sort
    3. ข้อมูลขนาดกลาง: Shell Sort เหมาะสม
    4. ข้อมูลขนาดใหญ่และต้องการความเร็ว: Quick Sort หรือ Merge Sort
    5. ต้องการความเสถียร (Stability): Merge Sort หรือ Insertion Sort
    6. หน่วยความจำจำกัด: Quick Sort, Heap Sort (ไม่ได้กล่าวถึง)
    7. ต้องการความเร็วสม่ำเสมอ: Merge Sort

สรุป: การเลือกอัลกอริทึมการเรียงลำดับที่เหมาะสมขึ้นอยู่กับลักษณะของข้อมูล ข้อจำกัดด้านหน่วยความจำ และความต้องการด้านความเร็ว การเข้าใจหลักการทำงาน จุดแข็ง และจุดอ่อนของแต่ละวิธีจะช่วยให้สามารถเลือกใช้ได้อย่างเหมาะสมกับสถานการณ์ต่างๆ
